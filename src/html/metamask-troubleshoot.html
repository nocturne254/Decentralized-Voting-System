Common MetaMask Issues and Solutions

1 MetaMask Not Installed

If you don't have MetaMask installed, you'll need to install it first:

  1. Visit https://metamask.io/download/
  2. Choose your browser and follow the installation instructions
  3. Create a new wallet or import an existing one
  4. Refresh this page after installation

2 MetaMask Not Connected to Ganache

You need to configure MetaMask to connect to your local Ganache blockchain:

  1. Open MetaMask
  2. Click the network dropdown at the top (usually says "Ethereum Mainnet")
  3. Click "Add Network" or "Custom RPC"
  4. Enter the following details:
    • Network Name: Localhost 7545
    • New RPC URL: http://localhost:7545
    • Chain ID: 1337 (or 5777 depending on your Ganache configuration)
    • Currency Symbol: ETH
  5. Click "Save"
Note: Make sure Ganache is running before adding the network.

3 No Accounts Imported from Ganache

You need to import at least one account from Ganache into MetaMask:

  1. In Ganache, click on the key icon next to an account to reveal its private key
  2. Copy the private key
  3. In MetaMask, click your account icon in the top-right corner
  4. Select "Import Account"
  5. Paste the private key and click "Import"
Warning: Never use these imported accounts on the main Ethereum network. They should only be used for local development.

4 MetaMask is Locked

If MetaMask is locked, you'll need to unlock it:

  1. Click the MetaMask icon in your browser
  2. Enter your password
  3. Click "Unlock"

5 Reset MetaMask Account

If you're still having issues, try resetting your MetaMask account:

  1. Open MetaMask
  2. Click your account icon → Settings → Advanced
  3. Scroll down and click "Reset Account"
  4. Confirm the reset
Note: This will clear your transaction history but keep your accounts.

6 Check Ganache Configuration

Make sure Ganache is properly configured:

  1. Ganache should be running on port 7545
  2. Chain ID should be set to 1337 (or 5777)
  3. Auto-mining should be enabled
Tip: You can create a workspace in Ganache and save these settings for future use.

7 Check Truffle Configuration

Your truffle-config.js should match your Ganache settings:

module.exports = {
  networks: {
    development: {
      host: "127.0.0.1",
      port: 7545,
      network_id: "*" // Match any network id
    }
  }
};

8 Browser Issues

Try these browser-related fixes:

  • Clear your browser cache and cookies
  • Try a different browser
  • Disable other extensions that might interfere with MetaMask
  • Try using a private/incognito window
